## Architectural decisions

### Monorepo structure

RAPP runs four apps from a single repository: Website Web, Website Mobile, Builder Web, and Builder Mobile. All four share one set of dependencies, one build configuration, and one Vite dev server on port 3000. This eliminates the overhead of publishing shared packages between repos and allows cross-app changes in a single commit.

A custom Vite plugin (`multi-app-plugin.ts`) handles URL-based routing to determine which app to serve, while dynamic imports in `src/main.tsx` ensure each app loads only its own code.

### Zustand for state management

Zustand was chosen over Redux and React Context for three reasons: minimal boilerplate, excellent performance through selective re-renders, and built-in `persist` middleware that saves state to `localStorage`. This means conversation data and app specifications survive browser refreshes without any additional infrastructure. Each app maintains its own stores while sharing auth utilities through `@rapp/shared-web`.

### Vite as build tool

Vite provides near-instant hot module replacement during development and optimized production builds. A custom multi-app plugin extends Vite to serve all four apps from a single dev server, rewriting URL paths to the shared `index.html` entry point and resolving per-app `@/` import aliases. Shared dependencies (React, Zustand, React Router, Radix UI) are pre-bundled to prevent staggered dependency discovery and unnecessary page reloads during development.
