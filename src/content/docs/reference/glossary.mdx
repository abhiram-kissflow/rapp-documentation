---
title: Glossary
description: Key terms used throughout RAPP documentation
---

| Term | Definition |
|------|-----------|
| Blueprint | A complete specification for your app, covering roles, processes, forms, rules, and integrations. Generated by AI from your description. |
| BRD | Business Requirements Document — the formal name for a blueprint. |
| Builder | The visual editor where you refine your app's blueprint, chat with AI, and preview the result. Available at `/builder`. |
| Code splitting | A performance technique where each app's JavaScript is loaded only when needed, so visiting one app never downloads another app's code. |
| Context Analyzer | The Conversation Engine component that interprets your initial description, identifying the app type, likely users, and key requirements. |
| Conversation Engine | The AI system that interviews you about your app idea, asking smart questions to understand your needs. Lives in the Website app. |
| Dynamic import | A JavaScript feature that loads code on demand. RAPP uses dynamic imports in `src/main.tsx` to load only the requested app. |
| Dynamic Interview | An AI-driven questioning process that adapts based on your answers, asking only 3-5 targeted questions. |
| Git submodule | A pointer inside one Git repository to a specific commit in another. RAPP uses a submodule to include the KFDS component library. |
| KFDS | Kissflow Design System — the UI component library providing consistent, accessible interface elements across the Builder. |
| Monorepo | A single Git repository containing multiple applications. RAPP's monorepo holds four apps plus shared utilities. |
| Multi-App Plugin | A custom Vite plugin that routes browser requests to the correct app based on URL path and resolves per-app import aliases. |
| Path alias | A shortcut for import paths configured in Vite (e.g., `@kfds-web` resolves to the design system's web components folder). |
| Pattern | A type of app architecture. RAPP recognizes workflow, portal, data collection, and tracking patterns. |
| Playbook | Knowledge documents that teach the AI about different types of business applications and how to build them. |
| Question Planner | The Conversation Engine component that decides which questions to ask next based on what is already known. |
| Response Generator | The Conversation Engine component that crafts natural-language messages and questions for the user. |
| Sandbox | An isolated preview environment where you can see and interact with the app generated from your blueprint. |
| Spec | Short for specification — another name for a blueprint. |
| Voice Pipe | The bidirectional voice system that lets you speak with the AI instead of typing, powered by OpenAI's Realtime API. |
| Voice Proxy | A Node.js WebSocket server (`openai-voice-proxy/`) that bridges the browser and OpenAI's Realtime API for voice conversations. |
